Setting up MarcoPolo's actions

Actions are triggered when MarcoPolo decides to change your location.

Each action has the following properties:

  • A type,
  • A parameter for the type of action,
  • A location that it should be triggered for,
  • When it should be triggered (upon Arrival or Departure),
  • A delay between changing location and triggering the action, and
  • A description for human readability.

Most actions will be Arrival actions; that is, they will be triggered when you arrive at a location and MarcoPolo decides to switch to that location. The alternative is a Departure action, which is triggered when MarcoPolo decides to switch away from that action's location. In this case, the delay parameter will be the amount of time from the action being triggered until the location is switched.

DefaultPrinter actions

Actions of type DefaultPrinter change your computer's default printer. Its parameter is the name of the printer queue to switch to. The printer queue name for a given printer is viewable through the Printer Setup Utility (/Applications/Utilities/Printer Setup Utility.app), using the 'Show Info' button. It is usually the name of the printer, with spaces and hyphens replaced by underscores.

MailSMTPServer actions

Actions of type MailSMTPServer change the default SMTP server of your Mail.app accounts to whichever SMTP server (as listed in Mail's preferences) has a hostname equal to the action's parameter.

Open actions

Actions of type Open are the most general actions. Their parameter is the full (absolute) path to a file on your computer. Upon triggering, MarcoPolo will open that file: if it is an application, it will be launched; otherwise, it will be opened with that file's default application. This is roughly equivalent to double-clicking on that file in Finder.

ShellScript actions

Actions of type ShellScript allow you to execute arbitrary shell scripts using /bin/sh. Its parameter is the full path to the shell script.

ToggleBluetooth and ToggleWiFi actions

Actions of type ToggleBluetooth or ToggleWiFi allow you to control the power state of your computer's built-in Bluetooth and WiFi controllers. The acceptable parameters for each are on or off.
